# Zony

축제 현장에서 신뢰도 있는 실시간 정보를 얻기 어려운 문제를 해결하기 위해  
위치 인증 기반 실시간 익명 채팅 서비스를 제공하는 웹 애플리케이션입니다.

축제 반경 내에 있는 사용자만 채팅에 참여할 수 있도록 설계하여  
현장 이용자들 간 실시간 정보 공유와 소통이 가능하도록 했습니다.

# Problem Solving

## 1. API 호출 타이밍 문제로 인한 UI 불일치 해결

### 문제

페이지 진입 시 불필요한 API가 호출되거나  
오래된 데이터가 유지되어 UI와 실제 데이터가 불일치했습니다.

### 원인

- API 호출 시점이 컴포넌트마다 달라 상태 최신화가 일정하지 않았습니다.
- 캐싱 유지 시간과 리셋 방식이 통일되지 않았습니다.

### 해결

- 진입 시 자동 호출되는 API를 비활성화
- 실시간 데이터 사용 구간에서는 `resetQueries`로 이전 데이터 초기화
- 데이터 유지 시간을 상황별로 재조정

### 성과

데이터 갱신 흐름이 명확해졌으며  
화면 전환 시 UI 불일치 문제가 사라졌습니다.

---

## 2. 실시간 무한 스크롤 채팅 구현 (스크롤 위치 보정)

### 문제

메시지 로드 지점마다 스크롤이 튀고  
최상단 / 최하단 로직이 동시에 실행되어 API가 중복 호출되었습니다.

### 원인

비동기 메시지 로드 과정에서 `scrollHeight`가 순간적으로 변하며  
스크롤 위치를 안정적으로 보정하지 못했습니다.

### 해결

- `useRef`로 이전 `scrollHeight` 저장
- 스크롤 보정 전까지 API 재호출을 막는 boolean 로딩 플래그 추가
- 중간 메시지 탐색 중에는 자동 스크롤 비활성화

### 성과

스크롤 점프 없이 부드러운 무한 스크롤을 구현했으며  
불필요한 API 호출도 제거했습니다.

## 3. Mapbox 클러스터 분리 및 개별 포인트 커스텀 렌더링

### 문제

Mapbox 기본 레이어로는  
원하는 디자인의 클러스터와 개별 포인트를 동시에 구현하기 어려웠습니다.

### 원인

기본 클러스터 레이어는  
포인트 전체에 일괄 적용되는 구조이기 때문에  
복잡한 커스텀 디자인 적용에 한계가 있었습니다.

### 해결

- 클러스터는 기본 레이어 그대로 사용
- 클러스터에 속하지 않은 포인트만 추출하여 `Marker`로 별도 렌더링
- 줌 / 이동 이벤트에 따라 포인트 리스트를 동적으로 재생성

### 성과

포인트와 클러스터를 완전히 커스텀한 UI로 구현하여  
지도 가독성과 사용자 경험을 개선했습니다.
